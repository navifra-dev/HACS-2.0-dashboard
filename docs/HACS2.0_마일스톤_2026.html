개발 의존관계 및 주요 제약사항

  • [ACS 핵심 흐름] 지도 메시지 + VDA5050 통신 → 1대 시뮬 주행 (3월) → 미션 할당 (4월) → 5대 경로 계획 (5월) → 10~20대 확장 (6월) → 50대 부하 테스트 (7월)
  • [VDA5050] 처음부터 VDA5050 프로토콜로 개발 (마이그레이션 없음) — MQTT Status/Visualization/Order 메시지 기반
  • [로봇 스케일업] 1대 → 2대 → 5대 → 10~20대 → 50대 단계적 확장 — 매월 로봇 수 증가로 고객에게 진척 가시화
  • [PLC 연동] OPC UA 환경 구축(3월) → ACS PLC 연동(4월) → 대규모 150세션(5~6월) → 통합 서버(7월)
  • [ETA] ETA 추정 계산(5월) → ETA 학습/보정(6월) → ETA 진척률 UI(6월) — 미션 할당 이후 순차 개발
  • [H-PLC] Polling(과부하 위험) vs Pub/Sub(누락 위험) — 두 방식 모두 신뢰성 검증 필요 → 가상 PLC 환경에서 빠른 성능 검증
  • [H-PLC] H-ACS + H-PLC 단일 서버 운영 · H-PLC 2~3개 인스턴스 동시 운영 · 1,500 TAG (150 PLC × 10) 300~500ms 주기
  • [병렬] ACS / UI·UX / H-PLC / 기획은 독립적 병렬 개발 — 8월 통합·검증 단계에서 합류
